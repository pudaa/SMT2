# updates

## SMT 2.0.0：整体的重构

- 使用 PySide6 重构原有代码，将内存开销从 200MB 降低至 34MB（没有开启性能追踪时），有更好的对高分辨率的支持。不足是启动需要管理员权限。
- 增加了动画
- 实现了待办事项的拖动顺序切换

## SMT 2.0.1：优化待办事项的行为

- 用户按下 enter 的时候将会成功失焦，并且刷新列表
- 用户点击空白处时，将会自动销毁空白列表项和已完成列表项（缺点是不容易反悔了，可能之后需要加一个撤回操作吧）
- 添加了提示框以辅助长文本的显示（后续有意向改为右键唤出详细内容弹窗）
- 使用 QFontMetrics 优化文本显示，保留重点内容
- 意外收获：解决之前不使用管理员启动就无法置顶的问题

## SMT 2.0.2：标签系统上线

- 在待办事项中添加了标签系统，通过勾选标签来筛选待办事项
- 缺点是启动速度好像变慢了，可能和NPL的运行有关，不知道对内存占用影响大不大
- 后续还是改回了jieba实现，虽然没有spacy分词优秀，但是速度和兼容性都有所提升
- 麻烦的地方是，分词的词典占用内存很大，会导致运行内存占用达到185MB，赶上Java版的SMT了。解决方案是准备了一个利用正则表达式，基于词频统计来实现标签的提取。

## SMT 2.0.3：更加自由的自定义

- 将原本硬编码的文件路径和颜色信息改为配置文件设置
- 以此为基础添加配置面板，允许用户对自己的SMT进行自定义，打造自己的SMT
